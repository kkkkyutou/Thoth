# Change Decisions

This file records user decisions that change how New Thoth should be interpreted.

## Decisions

1. `NTH-CD-001` `[2026-06-27]`: The old Thoth plugin form is the final archived plugin line. It is no longer maintained. Archive release: `thoth-plugin-final-archive`. Archive branch: `archive/main-20260627`.
2. `NTH-CD-002` `[2026-06-28]`: New Thoth starts fresh. Old plugin code should be removed from the active working tree after enough design information is extracted.
3. `NTH-CD-003` `[2026-06-28]`: The New Thoth design authority is the core principles document plus three canonical design documents: high-level design, MVP user journey and engineering architecture.
4. `NTH-CD-004` `[2026-06-28]`: The hardest product problems are clarify quality and aggressive loop design, not ordinary software development.
5. `NTH-CD-005` `[2026-06-28]`: New Thoth should behave like a real private secretary for a high-authority user. It should remember context and ask only a few golden questions when human judgment is truly needed.
6. `NTH-CD-006` `[2026-06-28]`: A task loop must aggressively solve the previous unresolved problem. It must not mechanically repeat runs or only add incremental tests.
7. `NTH-CD-007` `[2026-06-28]`: The active repo reset used `GPL-3.0-only`, `npm workspaces`, package namespace `@thoth/*`, version `0.0.0`, and package skeletons marked `private: true`. The license part is superseded by `NTH-CD-017`.
8. `NTH-CD-008` `[2026-06-28]`: TUI must use OpenTUI. Node/Bun runtime choice is intentionally deferred to the first TUI spike.
9. `NTH-CD-009` `[2026-06-28]`: Old prompt assets are preserved as docs-only structured contract seeds, not as executable Python code.
10. `NTH-CD-010` `[2026-06-28]`: `CLAUDE.md` should be a symlink to `AGENTS.md` in this reset.
11. `NTH-CD-011` `[2026-06-29]`: New Thoth should later adopt a Paseo-like release and packaging pipeline. The intended shape is tag-triggered GitHub Actions rather than ordinary branch-push publishing: desktop installers are built with Electron Builder for macOS, Linux and Windows and uploaded to GitHub Releases; Android APK release builds use Expo/EAS and are uploaded to GitHub Releases; web/app deployment can use Cloudflare Pages or an equivalent hosted target; iOS should not be treated as a GitHub Release IPA self-install path and needs a separate TestFlight/App Store/EAS submit distribution decision.
12. `NTH-CD-012` `[2026-06-29]`: Thoth itself is not a harness tool and must not become a hidden LLM API wrapper. All AI and agent execution capability comes from configured providers: ACP adapters, harness runtimes, app-server sessions, official harness SDK/control surfaces or local harness CLIs. Thoth source owns process flow, routing judgment, preset prompt contracts, task authority, frozen acceptance, evidence and session records; it must not privately call general model inference APIs as a substitute for provider sessions.
13. `NTH-CD-013` `[2026-06-29]`: Thoth must not use local deterministic heuristics for zero-shot semantic routing. Any intelligent judgment about intent type, workspace context, ambiguity, clarification questions, route upgrade/downgrade or execution strategy must run inside a provider-backed session. The desktop composer should expose explicit Thoth-level controls for task mode (`quick`, `loop`), clarification strength and loop strength; local code may honor those explicit controls and run mechanical authority checks, but it must not silently infer task type from natural language.
14. `NTH-CD-014` `[2026-06-29]`: The chatbox composer controls are locked as `+`, Provider, Mode, Clarify and Loop. `+` only supports image attach and file upload under `10MB` for MVP. Scope is not a separate button; users reference scope with `@`. Provider contains provider/model/runtime settings including model id, thinking strength, permission mode and fast mode. Mode has two options: `Quick` and `Loop`. Clarify has five user-facing levels: `auto`, `no-ask`, `light`, `balance`, `deep`, and applies to both `Quick` and `Loop`; protocol enums may normalize them as `auto`, `no_ask`, `light`, `balanced`, `deep`. Loop has five user-facing levels: `auto`, `no-loop`, `light`, `balanced`, `endless`; it only applies when Mode is `Loop`, otherwise it is disabled/greyed out. `endless` is red, high-cost, and runs until the user manually stops it, while still respecting safety, permission, provider availability and resource boundaries.
15. `NTH-CD-015` `[2026-06-29]`: The business flow is updated around provider-native sessions. All visible provider output must stream through Thoth in real time. `Quick + Don't Bother Me` is a Paseo-like provider passthrough path. `Loop` uses a read-only Clarify provider session, then a frozen contract, then one PlanExec provider session using provider-native plan mode when available, then an independent Review provider session. Clarify may read workspace state and search the web but must not mutate files. After contract freeze, PlanExec provider clarification questions are auto-answered from the frozen contract or first recommended option and recorded; provider permission requests are still governed by permission policy. User-facing labels supersede the old names in `NTH-CD-014`: Clarify uses `Don't Bother Me` instead of `no-ask`; Loop uses `One Plan, One Do` instead of `no-loop` and `Run Until Stopped` instead of `endless`.
16. `NTH-CD-016` `[2026-06-30]`: Clarify is locked as a decision-tree walk rather than a questionnaire. It must ask the current highest-leverage root or branch decision first, after resolving discoverable facts through read-only investigation. Clarify provider output is split into visible stream text and structured interaction candidates. Visible provider text streams to clients in real time, but clarification cards render only after Thoth validates the structured candidate. Invalid card candidates, schema errors and repair prompts are hidden from the user; the daemon sends a concise repair report back into the same provider session and asks it to regenerate the same card for the same tree node. Client UI keeps a lightweight defensive validator, while the daemon owns canonical validation and repair authority.
17. `NTH-CD-017` `[2026-06-30]`: New Thoth switches to `AGPL-3.0-or-later`. Implementation seed import uses only the latest GitHub upstream source from `getpaseo/paseo`; the local dirty checkout is ignored. Raw upstream material lives only in ignored `.agent-os/upstreams/paseo/`; tracked seed material may live in `_paseo/` directories and may temporarily be broken. Multica source code must not be copied. Voice, audio, speech and dictation material is excluded from raw cache and tracked seeds.
18. `NTH-CD-018` `[2026-06-30]`: User approved promoting all tracked `_paseo` implementation seeds into formal package source trees and deleting `_paseo`. Package/config material may be adopted broadly, but formal Thoth package identity must remain `@thoth/*`, `private: true`, `AGPL-3.0-or-later`, version `0.0.0`, root `workspaces: ["packages/*"]` and the 10 package boundary. `packages/app/highlight` remains nested and no `packages/highlight` workspace is created. Voice/audio/speech/dictation product capability remains out of scope. Broken imports, scripts, dependencies and type errors are expected until dependency and compile triage.
19. `NTH-CD-019` `[2026-06-30]`: First-day development infrastructure is required before feature development. Root npm scripts are the standard validation and packaging entrypoints. Node is locked to `24.14.0`, npm to `11.9.0`, formatting/linting use `oxfmt` and `oxlint`, foundation green packages are `packages/app/highlight`, `packages/relay`, `packages/protocol` and `packages/client`, typecheck uses `tsc --noEmit`, Android Debug APK is a real local artifact built with project-local JDK/Android SDK/Gradle cache under `.dev/`, iOS scripts must skip/fail clearly on Linux, and no GitHub Actions, push, publish or release is authorized in this round. Root `.npmrc` defaults to `ignore-scripts=true`, `audit=false` and `fund=false`; native/toolchain work must be performed by explicit root scripts instead of package install lifecycle scripts.
20. `NTH-CD-020` `[2026-06-30]`: The dev UI entry for Thoth I must be the same user experience as the current releasable full UI. It may be a development build using local daemon/provider/runtime configuration, but it must not be a separate mock, debug-only, reduced or agent-facing UI. Humans use this UI for real dogfood, review and experience testing. Agents validate repository code through standard unit/typecheck/build/gate commands, not by treating manual UI exploration as the normal automated acceptance path.
21. `NTH-CD-021` `[2026-07-01]`: Relay protocol is now v3-only with daemon-first room registration and role-scoped capability tokens sent through `Sec-WebSocket-Protocol`, not URL query. The relay rejects legacy v1/v2, missing/malformed tokens, token query parameters and disallowed browser origins; stores only token hashes, expiry and connection metadata; keeps zero-knowledge ciphertext forwarding; and defaults to `relay.thoth.seeles.ai` / `app.thoth.seeles.ai`. Code4Agent is the intended deploy mirror for hosted relay, but Royalvice cannot push the required `apps/thoth-relay/wrangler.jsonc` and `_deploy-isolated.yml` changes because Code4Agent's active `protected-paths` push ruleset restricts `**/*/wrangler.jsonc` and `.github/**/*`; hosted preview deployment must therefore be done by Bot/admin or an allowed actor.
22. `NTH-CD-022` `[2026-07-01]`: Thoth runtime must be fully isolated from the local Paseo daemon. Thoth direct daemon defaults to `127.0.0.1:6688`; `127.0.0.1:6767` is reserved for the local Paseo/legacy daemon and must not be auto-probed, reused, stopped or restarted by Thoth. Human web review uses the real web UI at local `http://127.0.0.1:8082/` and current public mapping `http://180.76.242.105:8148/`. Relay deployment authority is superseded from the blocked Code4Agent mirror path to the independent private repository `SeeleAI/Thoth-Relay`; the current test endpoint is `relay.test.thoth.seeles.ai`, with broad test browser Origin allowance but mandatory relay v3 subprotocol tokens. Desktop packaging produces a local Linux AppImage; mobile packaging produces an Android Debug APK with package id `sh.thoth.debug` and no microphone permission.
