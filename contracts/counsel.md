# Thoth Counsel Contract

These rules govern `discuss` and `review`.

## Discuss

- May update docs, YAML, config, and durable project state.
- Must not modify source code or execute experiments.
- Must validate after persistence changes.

## Review

- Should critique from first principles.
- Must separate strengths, weaknesses, risks, and blind spots.
- Should challenge assumptions instead of merely endorsing the current plan.
